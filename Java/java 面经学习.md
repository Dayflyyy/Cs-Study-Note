# java 面经学习

## JDK JRE JVM的区别

+ JDK是java development kit 是开发者工具
+ JRE 是java runtime environment 是运行时环境，包括工具类等？
+ JVM是java虚拟机，提供底层的汇编编译

## Java中创建对象的方式

+ new
+ newinstance(),类方法，需要无参构造方法
+ constructor.newinstance()，获得类的构造器，可以使用有参构造或者无参构造
+ 反射机制
+ 序列化和反序列化，从字节流中重新构建对象

## == 和equal的区别

### ==

+ 判断基本数据类型时，==按值判断
+ 判断对象时，==按指向的内存地址判断

### equal

+ 按内存地址中的值判断

## & 和&& | 和||

double是短路求值

## java函数是传值还是传引用

+ 基本数据类型传值
+ 对象数据类型传引用的值，可以理解为新建了一个指针指向相同的内存区域

## java的序列化

就是在io/文件等将对象转化为字节流，方便传输，序列化和反序列化遵循相同的编码密钥

## java中的反射

先获取到类，再通过类实例化对象，能够动态决定运行时类型

### java反射的应用

1. jdbc的驱动加载
2. spring的xml配置加载
   1. 先将xml加载到内存中
   2. 然后获取各个类的字节码，获取各个类，然后动态加载属性

动态操作和配置类的属性，程序更灵活，提高扩展性。

缺点：性能比较低，动态解析和调用，有额外检查和处理的负担，有可能有安全问题。

## string 为什么要设置成为不可变类

被final关键词修饰，底层使用char数组（private/final）

1. 允许重用，允许哈希值缓存，提高性能表现
2. 线程安全
3. 安全性和可靠性

可以通过反射改变string中value中的值。
